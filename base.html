1 ¿Qué es una Base de Datos?

Una Base de Datos (BD) es una colección organizada y estructurada de información o datos que se almacena electrónicamente en un sistema informático. Permite guardar información a largo plazo y acceder a ella rápidamente.

Imagina una hoja de cálculo gigante de Excel, pero optimizada para ser leída por miles de personas y servidores al mismo tiempo sin colapsar.

Ejemplo Visual de una Tabla (Clientes)

ID (Llave Primaria) Nombre (Columna) Correo (Columna)
1 Carlos López carlos@mail.com
2 Sofía Castro sofia@mail.com
Cada fila completa representa un Registro (o Tupla)

2 ¿Qué es un DBMS / SGBD?

Un **Sistema de Gestión de Bases de Datos (SGBD)** o **DBMS** (Database Management System) es el software que sirve de interfaz entre la base de datos física y los usuarios o aplicaciones.

SGBD Relacionales (SQL)

Almacenan información en tablas conectadas entre sí mediante llaves. Son robustas y estructuradas.

Ejemplos: PostgreSQL, MySQL, SQLite
SGBD No Relacionales (NoSQL)

Almacenan información en formatos flexibles como JSON, grafos u objetos clave-valor.

Ejemplos: MongoDB, Redis, Cassandra

2 El Modelo Relacional y el lenguaje SQL

El **Modelo Relacional** organiza los datos en tablas estructuradas (relaciones). Para interactuar con estas tablas usamos un lenguaje estándar llamado SQL (Structured Query Language) o Lenguaje de Consulta Estructurada.

Las operaciones se dividen comúnmente en dos tipos de sentencias principales:

DML (Manipulación de Datos)
  • SELECT - Consultar/Buscar datos
  • INSERT - Insertar nuevos registros
  • UPDATE - Modificar datos existentes
  • DELETE - Borrar registros
DDL (Definición de Datos)
  • CREATE TABLE - Crear nuevas tablas
  • ALTER TABLE - Modificar estructuras
  • DROP TABLE - Eliminar una tabla completa

3 Claves (Keys) e Integridad Referencial

Para evitar la inconsistencia de los datos, las tablas del modelo relacional se vinculan entre sí por medio de columnas especiales denominadas **llaves**:

Llave Primaria (Primary Key - PK)

Un identificador único para cada registro en una tabla. No puede repetirse y no puede ser nulo. Por ejemplo: `id` o `cedula`.

Llave Foránea (Foreign Key - FK)

Una columna de una tabla que hace referencia a la Llave Primaria de otra tabla. Es la responsable de crear las relaciones de datos (Ejemplo: `cliente_id` en una tabla de `Pedidos`).
