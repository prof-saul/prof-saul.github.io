Módulo de Criptografía

Nivel Principiante: El Cifrado César

La criptografía es el arte de escribir mensajes secretos. El Cifrado César es una de las técnicas más antiguas y simples, usada por Julio César. Es un cifrado por sustitución en el que cada letra del texto original es reemplazada por otra letra que se encuentra un número fijo de posiciones más adelante en el alfabeto.

Demo: Máquina Enigma César

Nivel Intermedio: Criptografía Moderna (Simétrica vs Asimétrica)

El Cifrado César es fácil de romper. Hoy usamos algoritmos matemáticos complejos.

  • Simétrica (Ej. AES): Usa la misma llave para cifrar y descifrar. Es como cerrar una caja con una llave y darle una copia exacta a tu amigo. Es muy rápido.
  • Asimétrica (Ej. RSA): Usa dos llaves matemáticas ligadas: una Pública (que compartes con todos para que te envíen mensajes) y una Privada (que solo tú tienes para descifrarlos).

Demo: Cifrado AES (Advanced Encryption Standard)

Este es el estándar utilizado por gobiernos y bancos actualmente.

Nivel Avanzado: Funciones Hash y Firmas Digitales

A diferencia del cifrado, un Hash es unidireccional. No se puede "descifrar". Convierte cualquier cantidad de datos en una cadena de texto de longitud fija. Si cambias una sola letra del archivo original, el Hash cambia por completo (Efecto Avalancha). Sirve para verificar la integridad de los archivos y guardar contraseñas de forma segura en bases de datos.

Módulo de Redes

Nivel Principiante: ¿Cómo hablan las computadoras?

Para que dos computadoras se comuniquen, necesitan una dirección, igual que las casas postales. Esto es la Dirección IP (Ej. 192.168.1.5). Además, existen dos tipos principales de redes:

  • LAN (Local Area Network): La red de tu casa u oficina. Los dispositivos se conectan a un Router.
  • WAN (Wide Area Network): Una red inmensa. Internet es la WAN más grande del mundo.

Nivel Intermedio: El Modelo OSI y Protocolos

Los datos no viajan de golpe, se dividen en paquetes. El Modelo OSI describe 7 capas por las que pasa un dato antes de salir por el cable o WiFi.

Los protocolos más importantes de transporte son:

  • TCP (Transmission Control Protocol): Garantiza que el paquete llegue. Es seguro y ordenado (usado en la web, correos). Es como una llamada telefónica.
  • UDP (User Datagram Protocol): Es rápido pero no garantiza la entrega (usado en streaming de video, juegos online). Es como gritar un mensaje.

Nivel Avanzado: DNS, Routing y Sniffing

DNS (Domain Name System): Es la agenda telefónica de internet. Traduce nombres humanos (www.google.com) a direcciones IP (142.250.190.46) para que las máquinas lo entiendan. Si un atacante envenena el DNS (DNS Spoofing), puede redirigirte a una página falsa aunque escribas la URL correcta.

Packet Sniffing: Dado que la información viaja en paquetes, si estás en una red Wi-Fi pública sin cifrar (sin HTTPS/VPN), un atacante puede usar herramientas (como Wireshark) para "escuchar" el aire y capturar tus contraseñas en texto plano.

Regla de oro: Nunca realices transacciones bancarias ni ingreses contraseñas críticas en redes Wi-Fi públicas abiertas sin usar una VPN.

Módulo de Seguridad

Nivel Principiante: El Eslabón más débil (El Humano)

El 90% de los ciberataques comienzan por errores humanos.
Ingeniería Social y Phishing: Los hackers envían correos o mensajes que parecen legítimos (de tu banco, de Netflix) para engañarte y que entregues tus credenciales.

Contraseñas: Una contraseña como "123456" o "maria1990" se crackea en milisegundos.

Nivel Intermedio: Defensas Perimetrales y Malware

Malware: Es un software malicioso. Incluye Virus (infectan archivos), Gusanos (se propagan por la red sin interacción), Troyanos (se disfrazan de programas útiles) y Ransomware (secuestran tus archivos cifrándolos y piden rescate).

Firewall (Cortafuegos): Es un guardia de seguridad entre tu red interna y el internet. Bloquea conexiones no autorizadas basándose en reglas predefinidas (Ej. "Bloquear todo el tráfico entrante al puerto 445").

Nivel Avanzado: Ataques Web y Vulnerabilidades

Las aplicaciones web (como Facebook o tu banco) sufren de ataques específicos:

  • SQL Injection (SQLi): El atacante introduce código de base de datos en un campo de texto (como el login) para engañar al sistema y acceder, modificar o borrar la base de datos entera.
  • Cross-Site Scripting (XSS): El atacante inyecta código JavaScript malicioso en una página web vulnerable. Cuando un usuario normal visita la página, el código se ejecuta en su navegador, robando sus cookies de sesión.
  • Man-In-The-Middle (MITM): El atacante se sitúa entre el usuario y el servidor. El usuario cree que habla con el servidor, y el servidor cree que habla con el usuario, pero el atacante lee y puede modificar toda la conversación.
